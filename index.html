THE 5 MCP TOOLS
fetch_patient_context(patient_id)
Reads comprehensive FHIR R4 clinical record. Fetches conditions, medication history, labs, procedures, allergies — all structured for PA analysis.
→ conditions, medications, labs, procedures, allergies
lookup_pa_criteria(drug_name, indication?)
Retrieves payer PA requirements from 12-drug criteria database. Falls back to LLM-generated criteria for any drug not in the database.
→ required_criteria, step_therapy, icd10_codes, guidelines
score_clinical_match(patient_context, pa_criteria)
Core intelligence — LLM analyzes FHIR data against each PA criterion. Returns score, matched/missing criteria, step therapy evidence, clinical flags.
→ score/100, recommendation, matched[], missing[], flags[]
draft_pa_letter(drug, criteria, match, patient, ...)
Generates complete 5-paragraph PA justification letter. Grounded in FHIR data — never invents clinical facts. Auto-detects urgency for expedited review.
→ letter (full text), score, missing_criteria, is_urgent
draft_appeal_letter(drug, denial_reason, ...)
Generates firm 6-paragraph formal appeal. Rebuts denial reason point-by-point, cites clinical guidelines, quantifies patient safety risk, demands peer-to-peer.
→ appeal_letter, key_arguments[], peer_to_peer_requested
